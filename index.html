CPU Scheduling Algorithms

CPU Scheduling decides which process runs and when. CPU is the most important part of a computer.

Diagram Parts:
Process → Program waiting for CPU
Queue → Line of waiting processes
CPU → Executes instructions

FCFS – First Come First Serve

Processes execute in the order they arrive.

P2 (3s)
P3 (2s)
CPU
Real Life Example:
ATM queue – one person takes long time, others must wait.
Difficult Word:
Convoy Effect → One long process delays all others.
Theory:
  • Non-preemptive
  • Simple
  • High waiting time
  • Used in batch systems

SJF – Shortest Job First

CPU selects process with smallest burst time.

P2 (2)
P3 (1)
Real Example:
Teacher checks small homework first to finish quickly.
Difficult Word:
Starvation → A process waits very long and may never run.
Theory:
  • Minimum average waiting time
  • Needs burst time prediction
  • Risk of starvation

Conclusion

Different scheduling algorithms solve different problems. Choosing the right one improves system performance.
